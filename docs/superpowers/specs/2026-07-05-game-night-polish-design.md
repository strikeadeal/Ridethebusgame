# Game-Night Polish — Ride the Bus PWA

## Context

Users report the game "lacks the features of an enterprise-level PWA." The app already has the structural PWA features (offline service worker, resume/persistence, wake lock, installability) — what it lacks is *feel*. This round adds the sensory polish layer players notice every round: card animations, sound, a celebratory finish, and shareable results. User chose the **library-assisted** approach (framer-motion, howler.js, canvas-confetti) and **text-summary** sharing.

Constraints: iPhone Safari (installed to home screen), offline-first, engine (`src/engine/`) stays pure and untouched — all work lives in `src/ui/` and app shell.

## New dependencies

- `framer-motion` — card flip/deal animations, staggered reveals
- `howler` — audio playback with iOS unlock handling
- `canvas-confetti` — game-over celebration

## Design

### 1. Card animations (framer-motion)

- Wrap `src/ui/components/CardView.tsx` in a motion container:
  - **Flip reveal**: face-down → face-up is a 3D `rotateY` spring flip (two-sided card: back face + front face).
  - **Deal-in**: newly mounted cards slide/scale in from slightly above with a spring.
  - **Tap feedback**: `whileTap` scale on tappable cards.
- Pyramid rows (`PyramidScreen.tsx`): current-row reveal uses a small stagger.
- Respect `prefers-reduced-motion`: use framer-motion's `useReducedMotion` to collapse to instant transitions.
- Keep the `card`/`card-down`/`card-red` CSS classes; animation layer sits on top so styling and tests (`data-testid`) are unchanged.

### 2. Sound (howler) + mute

- New module `src/ui/sound.ts`: thin howler wrapper exposing `play('deal' | 'flip' | 'correct' | 'wrong' | 'rowAdvance' | 'busStart' | 'fanfare')`, plus `muted` state persisted to localStorage (same pattern as `src/ui/storage.ts`).
- Sound assets: short WAV files generated by a new `scripts/generate-sounds.mjs` (offline synthesis — license-free, self-contained, mirrors the existing `scripts/generate-icons.mjs` pattern) into `public/sounds/`, committed. `npm run sounds` script added.
- Service worker precache: extend `vite-plugin-pwa` glob in `vite.config.ts` to include `sounds/*.wav` so audio works offline.
- iOS audio unlock: howler auto-unlocks on first touch; silent-switch caveat accepted (Web Audio respects the ringer switch — nothing to do).
- Mute toggle: small speaker icon button rendered in the `Scoreboard` header row (visible during play) and on `HomeScreen`; state read/written via `sound.ts`.
- Screens call `play()` at engine-action dispatch points (correct/wrong guess in `Phase1Screen`/`BusScreen`, reveals in `PyramidScreen`, game over).

### 3. Celebratory game-over (canvas-confetti)

- `GameOverScreen.tsx`: on mount, fire a confetti burst themed to the felt-table palette (gold/ivory/brass, not rainbow), plus `fanfare` sound.
- Scoreboard rows stagger in with framer-motion; top drinker row gets its existing `score-top` highlight after the stagger lands.
- Skipped when `prefers-reduced-motion`.

### 4. Share results

- New module `src/ui/share.ts`:
  - `buildShareText(state, stage): string` — pure function, emoji scorecard, e.g.:
    `🚌 Ride the Bus — Dave escaped after 3 attempts` + per-player `🍺` lines.
  - `shareResults(text)`: `navigator.share` when available; fallback `navigator.clipboard.writeText` + return value so UI can show a "Copied!" toast.
- `GameOverScreen.tsx` gains a "Share results" button between the scoreboard and Play again.

## Testing

- Unit (Vitest): `buildShareText` output, sound-module mute persistence (mock localStorage + Howler).
- E2E (Playwright): mute toggle persists across reload; share button with `navigator.share` stubbed shows the copied/shared state; existing full-game spec must still pass (animations must not break `data-testid` selectors — set framer-motion transitions near-instant under reduced-motion, and Playwright runs with reduced motion emulated where needed).
- `npm run build` clean; deploy pipeline (test-gated Pages workflow) unchanged.

## Files touched

- New: `src/ui/sound.ts`, `src/ui/share.ts`, `scripts/generate-sounds.mjs`, `public/sounds/*.wav`, tests for sound/share.
- Modified: `CardView.tsx`, `Phase1Screen.tsx`, `PyramidScreen.tsx`, `BusScreen.tsx`, `BusRevealScreen.tsx`, `GameOverScreen.tsx`, `Scoreboard.tsx`, `HomeScreen.tsx`, `vite.config.ts`, `package.json`, `src/styles.css`.
